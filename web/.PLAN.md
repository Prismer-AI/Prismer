# 移动端重构实施计划

> 基于 `docs/mobile/` 文档体系的全面重构
> 当前状态: 旧 4-tab 结构 → 新 3-tab 结构 (Command/Files/Me)

---

## 概述

### 现有代码分析

**当前移动端结构 (v1 - 待废弃)**:
- `src/app/mobile/` - 4-tab 结构 (Discovery/Assets/Chat/Settings)
- `MobileTabBar.tsx` - 旧导航结构
- `page.tsx` - 微信风格的会话列表
- `chat/store.ts` - 独立的聊天 store (与桌面端不兼容)
- `MobileChat.tsx` - 复用桌面端 workspace 组件

**主要问题**:
1. 导航结构与 v2.0 设计不符 (4-tab → 3-tab)
2. 聊天 store 与桌面端分离，数据不同步
3. 缺少 Command 模块的任务监控功能
4. 缺少 Files 模块的文件管理功能
5. Settings 页面需重构为 Me 模块

## 技术决策

### 1. 数据层策略

**决策**: 最大化复用桌面端 store，避免数据分离

| 模块 | 数据来源 | 说明 |
|------|---------|------|
| Command | `workspaceStore` | messages, tasks, participants |
| Files | `assetStore` | 文件列表, 上传状态 |
| Me | `authStore` | 用户信息 |

### 2. WebSocket 连接

**决策**: 复用 `useMobileAgent` hook

```typescript
// Command 模块使用
const { messages, sendUserMessage } = useMobileAgent({
  sessionId: 'default',
  serverUrl: getAgentServerUrl(),
});
```

### 3. 导航模式

**决策**: 使用 Next.js App Router + 底部 Tab 导航

- 主导航: BottomTabBar (Command/Files/Me)
- 子页面: MobileHeader 返回按钮
- 模态: BottomSheet

### 4. 组件复用策略

| 组件 | 复用方式 |
|------|---------|
| MessageList | 直接复用, 调整样式 |
| ChatInput | 直接复用, 添加移动端扩展 |
| SiriOrb | 直接复用 |
| shadcn/ui | 直接复用, 调整尺寸 |

## 风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 桌面端 store 变更影响移动端 | 中 | 添加移动端适配层 |
| WebSocket 连接不稳定 | 低 | 复用已验证的 useMobileAgent |
| 键盘弹出布局问题 | 中 | 使用 100dvh + keyboard-aware |
| iOS 安全区适配 | 低 | 复用已验证的 SafeAreaProvider |
